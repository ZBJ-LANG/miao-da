## 四、MySQL 表结构

```sql
-- ============================================
-- 1. 用户表
-- ============================================
CREATE TABLE users (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID',
    openid VARCHAR(100) UNIQUE COMMENT '微信openid',
    nickname VARCHAR(100) COMMENT '用户昵称',
    profile JSON COMMENT '体型档案: {height, weight, skin_tone, style}',
    preferences JSON COMMENT '偏好设置: {colors[], avoid_styles[]}',
    budget JSON COMMENT '预算配置: {monthly, max_single}',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- ============================================
-- 2. 商家商品表 (从API一次性导入)
-- ============================================
CREATE TABLE products (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID',
    source VARCHAR(50) COMMENT '来源: taobao/jd/weipinhui',
    source_id VARCHAR(100) COMMENT '原始商品ID',
    name VARCHAR(255) COMMENT '商品名称',
    category VARCHAR(50) COMMENT '主类别: 上衣/裤子/裙子/鞋子',
    sub_category VARCHAR(50) COMMENT '子类别: 衬衫/T恤/连衣裙',
    color VARCHAR(50) COMMENT '颜色',
    style VARCHAR(50) COMMENT '风格: 商务/休闲/运动',
    season VARCHAR(20) COMMENT '季节: 春夏/秋冬/四季',
    price DECIMAL(10,2) COMMENT '价格',
    image_url VARCHAR(500) COMMENT '商品图片URL',
    product_url VARCHAR(500) COMMENT '购买链接',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_category (category),
    INDEX idx_style (style),
    INDEX idx_color (color),
    INDEX idx_source (source)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商家商品表';

-- ============================================
-- 3. 虚拟衣橱表 (用户收藏的搭配)
-- ============================================
CREATE TABLE saved_outfits (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID',
    user_id VARCHAR(36) NOT NULL COMMENT '用户ID',
    name VARCHAR(100) COMMENT '搭配名称: 商务出差搭配',
    occasion VARCHAR(50) COMMENT '场合: 商务/休闲/约会',
    weather VARCHAR(50) COMMENT '天气场景',
    style VARCHAR(50) COMMENT '风格: 商务/休闲',
    rating INT DEFAULT 0 COMMENT '用户评分 1-5',
    is_favorite BOOLEAN DEFAULT FALSE COMMENT '是否收藏',
    outfit_image VARCHAR(500) COMMENT '生成的模特试穿图URL',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user (user_id),
    INDEX idx_style (style),
    INDEX idx_occasion (occasion)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='虚拟衣橱表';

-- ============================================
-- 4. 虚拟衣橱商品详情表
-- ============================================
CREATE TABLE saved_outfit_items (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID',
    outfit_id VARCHAR(36) NOT NULL COMMENT '搭配ID',
    product_id VARCHAR(36) COMMENT '商品ID',
    product_name VARCHAR(255) COMMENT '商品名称',
    product_price DECIMAL(10,2) COMMENT '价格',
    product_image_url VARCHAR(500) COMMENT '商品图片URL',
    product_url VARCHAR(500) COMMENT '购买链接',
    sort_order INT DEFAULT 0 COMMENT '排序: 1=上衣, 2=裤子, 3=鞋子',
    
    FOREIGN KEY (outfit_id) REFERENCES saved_outfits(id) ON DELETE CASCADE,
    INDEX idx_outfit (outfit_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='虚拟衣橱商品详情表';

-- ============================================
-- 5. 对话历史表
-- ============================================
CREATE TABLE conversations (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID',
    user_id VARCHAR(36) NOT NULL COMMENT '用户ID',
    session_id VARCHAR(100) COMMENT '会话ID',
    messages JSON COMMENT '消息列表: [{role, content, time}]',
    context JSON COMMENT '场景上下文: {city, weather, last_intent}',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_session (session_id),
    INDEX idx_user (user_id),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='对话历史表';
```

## 五、Neo4j 知识图谱存储格式

### 节点定义
```cypher
// 商品节点
(:Product {id, name, price, category, color, style})

// 类别节点
(:Category {name: "上衣"})
(:Category {name: "裤子"})
(:Category {name: "裙子"})
(:Category {name: "鞋子"})

// 颜色节点
(:Color {name: "蓝色"})
(:Color {name: "黑色"})
(:Color {name: "白色"})

// 风格节点
(:Style {name: "商务"})
(:Style {name: "休闲"})
(:Style {name: "运动"})

// 场合节点
(:Occasion {name: "商务"})
(:Occasion {name: "约会"})
(:Occasion {name: "日常"})
```

### 关系定义
```cypher
(:Product)-[:类别]->(:Category)
(:Product)-[:颜色]->(:Color)
(:Product)-[:风格]->(:Style)
(:Product)-[:适合场合]->(:Occasion)
```

## 九、完整数据流 - 商品搜索

```
用户输入: "帮我找一件蓝色商务衬衫"

    │
    ▼
Step 1: 意图识别 (服装推荐/商品搜索/虚拟试穿) (商品搜索)
    │
    ▼
Step 2: 从MySQL提取用户档案 ◄直接从数据库提取
    │
    ▼
Step 3: 提取搜索关键词
    │   - 类别: 上衣/衬衫
    │   - 颜色: 蓝色
    │   - 风格: 商务
    │
    ▼
Step 4: 并行检索 (Neo4j + Milvus) ◄不调用商品API
    │
    │   ├── Neo4j: 根据类别+颜色+风格查询商品
    │   └── Milvus: 向量相似度检索
    │
    ▼
Step 5: 组合搜索结果
    │   - 结果融合 (RRF算法)
    │   - 取TOP 20商品
    │
    ▼
Step 6: 固定模板组装
    │
    ▼
Step 7: 用户点击 "收藏加购" → 存入MySQL虚拟衣橱
```

### 商品搜索详细流程

```
Step 1: 意图识别
    │
    ▼
Step 2: 提取搜索条件
    │   ┌─────────────────────────────────────────────┐
    │   │ 从用户输入提取:                              │
    │   │ - 类别关键词: 上衣, 衬衫, T恤, 裤子...      │
    │   │ - 颜色关键词: 蓝色, 黑色, 白色...          │
    │   │ - 风格关键词: 商务, 休闲, 运动...          │
    │   │ - 价格关键词: 便宜, 贵, 200元以内...       │
    │   │ - 季节关键词: 春夏, 秋冬...               │
    │   └─────────────────────────────────────────────┘
    │
    ▼
Step 3: MySQL精确查询
    │   SELECT * FROM products 
    │   WHERE category='衬衫' AND color='蓝色' AND style='商务'
    │   LIMIT 50
    │
    ▼
Step 4: Milvus向量检索 (如有模糊描述)
    │   - 如果用户输入"显瘦的"、"百搭的"等模糊描述
    │   - 转为向量进行相似度检索
    │
    ▼
Step 5: 结果排序与融合
    │   - 按相关性排序
    │   - 按价格/销量筛选
    │
    ▼
Step 6: 返回结果
```

### 商品搜索 vs 穿搭推荐 对比

| 对比项 | 商品搜索 | 穿搭推荐 |
|--------|----------|----------|
| 用户意图 | 找特定商品 | 找搭配方案 |
| 检索方式 | 精确筛选为主 | 向量相似为主 |
| 结果数量 | TOP 20商品 | 3-5件搭配 |
| 是否生成模特图 | 可选 | 必须 |
| 是否支持收藏 | 是 | 是 |


## 十四、数据存储总结

| 存储 | 数据 | 说明 |
|------|------|------|
| MySQL | users, products, saved_outfits, saved_outfit_items, conversations | 持久化存储 |
| Neo4j | 商品三元组 (类别/颜色/风格/场合) | 结构化查询 |
| Milvus | 商品向量 (CLIP编码) | 相似度检索 |
| LangGraph State | 当前会话messages+context | 临时，会话结束清除 |
